2. "원영님이 하실 일" 섹션 따라 터미널 설정
3. `=== BOOTSTRAP PROMPT ===` ~ `=== BOOTSTRAP PROMPT 끝 ===` 사이 전체 복사 → 터미널에 붙여넣기

## 각 터미널이 '자체 검증' 하는 법

모든 터미널은 코드·문서 수정 후 **Task 도구** 로 서브에이전트 호출해서 **제3자 리뷰** 받음:

1. 작업 완료
2. `self-review.md` 10개 체크리스트 통과
3. **Task 호출 → 제3자 리뷰어** → 🟢/🟡/🔴 판정
4. 🟡 이상이면 **수정 후 재검증** (🟢 받을 때까지)
5. PR 생성

즉, **원영님이 일일이 검사 안 해도** 터미널이 스스로 2단계 검증함.

---

## 터미널끼리 부딪히면?

- 모든 작업은 **티켓 락**: 한 티켓은 한 터미널만 잡음
- 같은 파일 건드리면 `.ai/BOARD.md` 확인 → 늦은 쪽이 대기
- 급하면 T1 에게 조정 요청
- 그래도 안 되면 원영님(cowork) 에 `.ai/FOR_USER.md` 🔴 추가

## 원영님 전용 명령어 (cowork 에서 저한테 말할 때)

- **"오늘 할 일"** → `.ai/FOR_USER.md` 최신본 보여드림
- **"BOARD"** → 4 터미널 상태 요약
- **"T1 뭐해?"** → T1 담당 티켓 + BOARD 상의 현황
- **"승인"** → 대기 중 PR 승인 처리 안내
- **"청소"** → GC 리포트 최신본 보여드림
- **"새 티켓 만들어줘: XXX"** → 티켓 초안 작성

## 문제 상황 대응

| 증상 | 원인 | 해결 |
|------|------|------|
| 터미널이 말 안 들음 | bootstrap 프롬프트 일부만 붙여넣음 | `=== BOOTSTRAP PROMPT ===` 사이 **전체** 재복사 |
| 린트 에러 | `npm install` 안 함 | 레포에서 `npm install && npm run prepare` |
| Task 호출 실패 | 모델이 Task 권한 없음 | `/model opus` 또는 `/model sonnet` 재선택 |
| 같은 파일 편집 충돌 | 티켓 락 무시 | BOARD.md 확인 → 한쪽 중단 |
| 원영님 답 기다림 | FOR_USER.md 🔴 항목 방치 | 답변 칸 채워주시면 즉시 진행 |

## 🔥 터미널 여는 건 원영님이 직접

저(cowork 오케스트레이터)는 원영님 로컬 컴퓨터에서 터미널을 열 수 없어요. 그래서:

1. 원영님이 터미널 프로그램(Terminal, iTerm, Warp 등) 엽니다
2. `cd` 해서 레포 폴더로 갑니다
3. `claude` 명령 실행합니다
4. `/model xxx` 로 모델 선택합니다
5. 저희가 만든 kit 의 `BOOTSTRAP PROMPT` 전체 복붙합니다
6. Enter

→ 이후 **터미널 내부의 Claude Code** 가 자기 역할대로 일합니다.
