## 배경

원영님이 로컬에서 `npm install` 실행 중 다음 메시지 확인:

> `2 high severity vulnerabilities` (breaking change 포함이라 `--force` 필요)

자동 검사기 (ESLint / husky) 설치 과정에서 딸려 온 의존성 중 2개가 취약점 보고됨. 이게 우리 앱 런타임에 직접 영향을 주는지, devDependencies 한정인지 진단 필요.

---

## 목표

1. **어떤 패키지의 어떤 취약점**인지 정확히 파악 (CVE 번호, 심각도, 영향 범위)
2. **우리 앱에 실제 노출되는지** 판정 (devDependencies only 면 런타임 무관)
3. **수정 경로** 결정:
   - `npm audit fix` (무해한 경우)
   - `npm audit fix --force` (breaking change 감수, 수동 회귀 테스트 필요)
   - 패키지 고정/교체 (재앙 시)
   - 수용 후 기록 (운영 미영향 확실한 경우)

## 실행 단계

1. **진단** (오케스트레이터 직접 또는 T1)
   - `cd itdasy-frontend-test-yeunjun-main && npm audit --json` 결과 수집
   - 각 취약점을 devDep / prodDep 분류
   - 우리 앱 파일에서 해당 패키지 import 여부 grep

2. **판정** (오케스트레이터)
   - 3개 경로 중 선택, 원영님에 1줄 설명으로 승인 요청

3. **집행** (선택된 경로에 따라)
   - `npm audit fix --force` 택했으면: T4 가 전체 린트 + 테스트 재실행, `cap sync` 확인
   - 패키지 고정/교체 택했으면: T1 이 plan 쓰고 T2 FE 코더가 실행

4. **검증**
   - `npm audit` 에서 high severity 0 확인
   - husky pre-commit 훅 여전히 작동 확인
   - 기존 ESLint 규칙 (T-002) 영향 없음 확인
