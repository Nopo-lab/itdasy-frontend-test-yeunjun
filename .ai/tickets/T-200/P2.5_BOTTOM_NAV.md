# T-200 P2.5 · 하단 네비(`.bn`) 이식

> **상태:** 🔴 원영님 재스코프 승인 대기
> **선결 조건:** `PROTOTYPE_V2_ALIGNMENT.md` 전체 승인
> **담당:** T1 Architect (플랜) → T2 FE Coder (코드)

## 왜 필요한가

프로토타입 v2 는 하단 고정 네비(`.bn`) 5탭 구조. 현 구현은 상단 `.nav` 7개 버튼. 원영님이 지적: "하단 네비게이션은 어디갔어?" 프로토 v2 와의 가장 큰 구조적 불일치이고, P2(홈)·P3(작업실) 재작업의 전제 조건.

## 범위 (IN)

1. `style-components.css` 에 `.bn`, `.bn-btn`, `.bn-btn.active` 추가 (프로토 v2 L182–192 그대로)
2. `index.html` 에 `<nav class="bn">` 하단 삽입 (프로토 v2 L342–363 마크업, `onclick` 만 `showTab(...)` 로 치환)
3. 기존 `<div class="nav">` 은 `style="display:none"` + `aria-hidden="true"` 처리 (JS 호환 보존용)
4. `body` 에 `padding-bottom: calc(96px + env(safe-area-inset-bottom))` 적용
5. 활성 탭 동기화: `showTab` 호출 시 `.bn-btn.active` 도 같이 토글 (기존 `.nav-btn.active` 로직 참조해 쌍둥이 코드 추가)

## 범위 (OUT)

- 아이콘 Lucide 통일 → P2.6 분리
- 탭명 변경 (작업실 → 만들기 등) → P2.7 분리
- 홈/작업실 탭 내부 레이아웃 재작성 → P2-re / P3-re 로 분리

## 동결 셀렉터 영향

- ✅ 보존: `id="nav"`, `.nav-btn`, `showTab()` 함수명, `tab-home`~`tab-finish` id
- ➕ 추가 허용: `.bn`, `.bn-btn`, `data-tab`

## 수용 기준

- [ ] 모바일 ~420px 에서 하단 5탭 네비가 safe-area 포함해 고정
- [ ] 5개 버튼 클릭 시 각 `tab-XXX` 컨테이너 표시 (스모크 테스트)
- [ ] 활성 탭이 `--brand` 핑크로 강조, 비활성은 `--text-subtle`
- [ ] 상단 `.nav` 는 화면에 안 보이지만 DOM 에는 존재
- [ ] 라이트/다크 양 모드 스크린샷 첨부 (`.ai/tickets/T-200/P2.5/{light,dark}.png`)
- [ ] 교차검증 체크리스트에 **프로토 v2 L342–363 마크업 구조 일치** 항목 포함

## 예상 변경 규모

- CSS: +12줄
- HTML: +22줄 (+ 기존 `.nav` 에 `display:none` 1줄)
- JS: `showTab` 에 `.bn-btn` 토글 로직 5~10줄 추가 (기존 `app-core.js` 또는 신규 패치 모듈)

## 위험 & 완화

| 위험 | 완화 |
|------|------|
| `showTab` 이 `.nav-btn` 만 토글 → 하단 네비 활성 표시 안됨 | `showTab` 에 `document.querySelectorAll('.bn-btn')` 순회 추가 |
| `body padding-bottom` 전역 영향 | `.tab` 내부 `margin-bottom` 조정 또는 `.phone` wrapper 신설 |
| `예약 🗓️` 버튼 접근 경로 사라짐 | 본 Phase 전에 P2.7 (탭명 결정) 필요 — 임시로 설정 메뉴에 옮기는 것 권장 |
