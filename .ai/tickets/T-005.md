```

```

로봇 청소기가 "여기 쓰레기 있어요" 사진만 찍고, 실제 치우기는 원영님이 "청소기 돌려!" 하는 방식.

## 목표

매주 월요일 한국시간 08:00, 자동으로 **"청소 보고서"** 를 GitHub Issue 로 만듬. 원영님이 Issue 댓글로 "예/아니오" 결정.

## Acceptance Criteria

- [ ] `scripts/gc-weekly.js` 작성 (의존성 추가 없음, 읽기 전용)
- [ ] 스크립트가 수행할 **관찰만**:
  1. 파일별 줄 수 측정 → CLAUDE.md 기재와 비교, 불일치 리스트 (자동 수정 X, 보고만)
  2. `grep -c "onclick=" *.html *.js` → 인라인 핸들러 수 추이
  3. `grep -o "itdasy_\\w*" *.js | sort | uniq -c` → localStorage 키 사용 빈도
  4. `grep -o "TD-[0-9]\\+" *.js` → 남은 TD 티켓 ID
  5. 500줄 초과 파일 리스트
  6. 사용 빈도 0 의심 파일 리스트 (import/참조 없음)
- [ ] 결과를 `.ai/gc-reports/YYYY-MM-DD.md` 에 저장
- [ ] GitHub Issue 자동 발행 (제목: `[GC Weekly] YYYY-MM-DD · N개 정리 후보`)
- [ ] Issue 본문은 **원영님용 쉬운 말**:
  ```
  이번 주 청소 후보 3개 찾았어요.
  
  1. [  ] `app-scheduled.js` 가 어디에서도 안 쓰이는 것 같아요. 지울까요?
  2. [  ] `CLAUDE.md` 에 "style.css 1383줄" 이라 적혀있는데 실제는 10줄. 문서 고칠까요?
  3. [  ] localStorage 키 `itdasy_hash_history` 가 작년부터 안 쓰인 흔적. 지울까요?
  
  결정: 각 항목에 [x] 표시하고 "승인" 댓글 달아주세요.
  에이전트가 자동으로 새 티켓 만들어서 실행합니다.
  ```
