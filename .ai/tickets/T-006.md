# T-006 · 고객센터 로그인 확인 옛날 방식 제거

## 🗣️ 원영님용 한 줄

**고객센터 기능**에서 로그인 확인 코드가 옛날 방식 그대로예요. 자동 검사기가 커밋 막아요. 1줄만 고치면 끝.

---

**상태:** READY TO START (T1 Architect 가 먼저 픽업 → T2 FE Coder 에게 위임 가능)
**Phase:** 1 (출혈 막기)
**신호등:** 🟡 노랑 (코드 1줄 수정, 계획→승인→실행)
**의존:** 없음 (T-003 후속, 독립 실행 가능)
**예상 소요:** 10분

## 배경

- T-003 에서 `app-push.js:42` 의 레거시 키 단독 체크를 `window.getToken()` 로 교체함
- 리뷰 과정에서 **`app-support.js:167` 도 동일한 버그** 발견됨
- 이전에 오케스트레이터가 "4개 키 방어 체크라 false positive" 로 오판정했음. 정정.
- T-002 의 ESLint `no-restricted-syntax` 규칙이 error 레벨이라, 이 파일 고치지 않으면 `npm run lint` 즉시 fail

## 현재 코드

```js
// app-support.js:164-168
if (!localStorage.getItem('itdasy_token::staging') &&
    !localStorage.getItem('itdasy_token::prod') &&
    !localStorage.getItem('itdasy_token::local') &&
    !localStorage.getItem('itdasy_token')) return;
```

체크: 4개 키 중 **하나라도 있으면 통과**. 하지만 마지막 `itdasy_token` (레거시) 는 `app-core.js` 마이그레이션 블록에서 이미 정리되므로 제거해도 안전.

## 수정 내용

```diff
- if (!localStorage.getItem('itdasy_token::staging') &&
-     !localStorage.getItem('itdasy_token::prod') &&
-     !localStorage.getItem('itdasy_token::local') &&
-     !localStorage.getItem('itdasy_token')) return;
+ // T-006: 레거시 키 itdasy_token 제거 + getToken() 통합
+ // app-core.js getToken() 이 _TOKEN_KEY 기반으로 현재 환경 토큰만 반환.
+ // 만료 체크도 포함되므로 수동 분기 불필요.
+ if (!window.getToken()) return;
```

## 수용 기준 (Acceptance Criteria)

1. `grep "'itdasy_token'" app-support.js` 결과 0건
2. `grep "'itdasy_token'" *.js` 결과는 `app-core.js` 와 `.eslintrc.js` 만 (마이그레이션 블록 + 차단 규칙)
3. `npm run lint` 실행 시 `no-restricted-syntax` 에러 0건 (warn 은 무관)
4. `app-support.js` 다른 로직 변경 없음 (줄 수 차이 ±5 이내)
